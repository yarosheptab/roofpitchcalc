## Version Warning

**This project uses Next.js 16 and Tailwind CSS v4** — both have breaking changes from their widely-known predecessors. Before writing any Next.js or Tailwind code, read the relevant guide in `node_modules/next/dist/docs/`. Your training data likely reflects older APIs.

## Commands

```bash
npm run dev          # Start dev server (localhost:3000)
npm run build        # Production build
npm run lint         # ESLint
npm run test         # Vitest (single run)
npm run test:watch   # Vitest watch mode
npm run test:coverage
```

There is no `npm run type-check` script — use `npx tsc --noEmit` if needed.

## Architecture

**Single-page tool + blog.** The homepage (`app/page.tsx`) is a server component that renders the interactive calculator, a static pitch reference table, FAQ, and recent blog posts. The calculator itself (`components/Calculator.tsx`) is a `'use client'` component.

### Calculation layer (`lib/calculations.ts`)

Pure functions, no side effects. Three entry points — `fromRiseRun`, `fromAngle`, `fromPitchRatio` — all normalise to a `pitchRatio` (X in X:12 notation) and call `buildResult`, which derives `angleDeg`, `slopePct`, `rafterMultiplier`, and `category`. `addRafterLength` is applied separately when the user provides an optional roof span. Tests live in `lib/calculations.test.ts` (Vitest, node env).

### Blog (`content/blog/` + `lib/blog.ts` + `app/blog/`)

MDX files in `content/blog/` are parsed server-side with `gray-matter`. `lib/blog.ts` exposes `getAllPostsMeta()` and `getPost(slug)`. Blog posts are rendered at `app/blog/[slug]/` using `next-mdx-remote`. MDX frontmatter fields: `title`, `description`, `date`, `category`, `author`, `readingTime`.

### Design system

Tailwind v4 with a custom `@theme` block in `app/globals.css`. Use these semantic tokens — **not** default Tailwind palette classes:

| Token | Value | Usage |
|---|---|---|
| `bg-bg` / `bg-surface` | `#F4F1EA` / `#FDFCF8` | Page vs card backgrounds |
| `text-primary` / `text-muted` | `#1B1814` / `#7A7168` | Body vs secondary text |
| `text-accent` | `#C0391E` | Accent/brand red |
| `border-border` | `#DDD8CC` | All borders |
| `text-data` | `#0C2340` | Numeric/data values |

Fonts loaded via `next/font/google` and exposed as CSS vars: `--font-sora` (sans), `--font-dm-serif` (serif/headings), `--font-geist-mono` (mono). Reference them via Tailwind's `font-sans`, `font-serif`, `font-mono` classes.

### SEO / Structured data

`lib/structured-data.ts` generates JSON-LD schemas. `components/JsonLd.tsx` injects them via `<script type="application/ld+json">`. The homepage renders `WebApplication`, `FAQPage`, and `BreadcrumbList` schemas. Blog pages add `Article` schema. `app/sitemap.ts` and `app/robots.ts` are auto-generated by Next.js conventions.

`NEXT_PUBLIC_SITE_URL` env var controls the canonical domain (defaults to `https://roofangler.com`). Copy `.env.local.example` to `.env.local` for local overrides.

### SVG diagram

`components/RoofDiagram.tsx` renders a live SVG roof cross-section that updates as the user types. The diagram reflects the computed pitch angle visually. The favicon (`app/icon.svg`) uses a similar angle-arc motif.
